# 算法导论4 分治

递推式（recurrence）是使用更小输入的项来描述当前项的等式或不等式。本章提供了三种求解递推式的方法，分别是替换法（substitution method），递归树法（recursion-tree method），主方法（master method）。

# 4.1 最大子数组问题

最大子数组问题：给定数组 A[1..n]，求 i 和 j 使得 A[i..j]（至少包含一个元素）之中所有元素之和最大。

# 分治算法

考虑使用分治策略求 A[low..high] 的最大子数组。首先使用分治法将数组等分，得到 A[low..mid] 和 A[mid + 1..high]。那么 A[low..high] 上的解能且只能是以下三种的一个：

  • 完全在 A[low..mid] 中。
  • 完全在 A[mid + 1..high] 中。
  • 横跨 A[low..mid] 和 A[mid + 1..high]。

前两种情况的解可由递归调用得到，因此仅需解决第三种情况。已知该子数组必须包含 A[mid 和 A[mid + 1]，所以可以从 mid 开始，向左找到总和最大的 A[i..mid]，向右找到总和最大的 A[mid + 1..j]，从而得到最终结果 A[i..j]。容易得到这一步的最坏情况运行时间是 Θ(n)\Theta(n)Θ(n)，其中 nnn 是 A 的长度。伪代码如下：

FIND-MAX-CROSSING-SUBARRAY(A, low, mid, high)
  // 向左找到总和最大的 A[i..mid]
  left-sum = - infinity
  sum = 0
  for i = mid downto low
    sum = sum + A[i]
    if sum > left-sum
      left-sum = sum
      max-left = i

  // 向右找到总和最大的 A[mid + 1..j]
  right-sum = - infinity
  sum = 0
  for j = mid + 1 to high
    sum = sum + A[j]
    if sum > right-sum
      right-sum = sum
      max-right = j

  return (max-left, max-right, left-sum + right-sum)

FIND-MAXIMUM-SUBARRAY(A, low, high)
  if high == low
    // 基本情况
    return (low, high, A[low])
  else
    // 递归情况
    mid = (low + high) / 2
    (left-low, left-high, left-sum) = FIND-MAXIMUM-SUBARRAY(A, low, mid)
    (right-low, right-high, right-sum) = FIND-MAXIMUM-SUBARRAY(A, mid + 1, high)
    (cross-low, cross-high, cross-sum) = FIND-MAX-CROSSING-SUBARRAY(A, low, mid, high)

    // 选出最大的
    if left-sum >= right-sum and left-sum >= cross-sum
      return (left-low, left-high, left-sum)
    else if right-sum >= left-sum and right-sum >= cross-sum
      return (right-low, right-high, right-sum)
    else
      return (cross-low, cross-high, cross-sum)

# Strassen 算法

Strassen 算法的关键在于让递归树稀疏一些。在上一节介绍的平凡分治算法中，递归树中的每个非叶节点有八个子节点；而在 Strassen 算法中仅有七个，这是通过增加一些额外的常数时间实现的。

Strassen 算法的详细步骤非常晦涩，其中隐含的数学关系太隐蔽。以下是 Strassen 算法的大致流程：

  1. 将矩阵 AAA、BBB 和 CCC 分割成 n/2×n/2n / 2 \times n / 2n/2×n/2 的子矩阵。这一步花费的时间是 Θ(1)\Theta(1)Θ(1)。
  2. 创建 101010 个大小为 n/2×n/2n / 2 \times n / 2n/2×n/2 的矩阵 S1,S2,…,S10S_1, S_2, \dots, S_{10}S1​,S2​,…,S10​，每个矩阵都是上一步中得到的子矩阵的和或差。这一步花费的时间是 Θ(n2)\Theta(n^2)Θ(n2)。
  3. 使用子矩阵和上一步得到的 101010 个矩阵，递归地计算七个乘法 P1,P2,…,P7P_1, P_2, \dots, P_7P1​,P2​,…,P7​。其中 PiP_iPi​ 的大小是 n/2×n/2n / 2 \times n / 2n/2×n/2。
  4. 通过 PiP_iPi​ 之间的加减得到 C11C_{11}C11​、C12C_{12}C12​、C21C_{21}C21​、C22C_{22}C22​，最终组合成 CCC。这一步花费的时间是 Θ(n2)\Theta(n^2)Θ(n2)。

先不谈算法的细节，现在可以得到运行时间的递推式：

T(n)={Θ(1)如果 n=17T(n/2)+Θ(n2)如果 n>1T(n) = \begin{cases} \Theta(1) & \text{如果 } n = 1 \\ 7 T(n / 2) + \Theta(n^2) & \text{如果 } n > 1 \end{cases} T(n)={Θ(1)7T(n/2)+Θ(n2)​如果n=1如果n>1​

# 4.3 替换法

替换法通过两个步骤求解递推式：

  1. 猜测解的形式。
  2. 通过数学归纳法求出常量的值，证明此解确实是该递推式的解。

可以通过替换法求得渐近上界和渐近下界。以求解渐近上界为例，原递推式：

T(n)=2T(n/2)+nT(n) = 2 T(n / 2) + n T(n)=2T(n/2)+n

这个递推式已经见到很多次了，可以猜测 T(n)=O(nlg⁡n)T(n) = O(n \lg n)T(n)=O(nlgn)。此时需要证明，当选择一个合适的正的常数 ccc 之后，可以使 T(n)⩽cnlg⁡nT(n) \leqslant c n \lg nT(n)⩽cnlgn 成立。假设它在 m<nm < nm<n 时总成立，那么令 m=n/2m = n / 2m=n/2，有 T(n/2)⩽c(n/2)lg⁡(n/2)T(n / 2) \leqslant c (n / 2) \lg(n / 2)T(n/2)⩽c(n/2)lg(n/2)。所以：

T(n)=2T(n/2)+n⩽2[c(n/2)lg⁡(n/2)]+n⩽cnlg⁡(n/2)+n=cnlg⁡n−cnlg⁡2+n=cnlg⁡n−cn+n⩽cnlg⁡n\begin{aligned} T(n) & = 2 T(n / 2) + n \leqslant 2 [c (n / 2) \lg(n / 2)] + n \\ & \leqslant c n \lg(n / 2) + n = c n \lg n - c n \lg 2 + n \\ & = c n \lg n - c n + n \leqslant c n \lg n \end{aligned} T(n)​=2T(n/2)+n⩽2[c(n/2)lg(n/2)]+n⩽cnlg(n/2)+n=cnlgn−cnlg2+n=cnlgn−cn+n⩽cnlgn​

当 c⩾1c \geqslant 1c⩾1 时最后一行成立。总结一下，上述过程证明了 c⩾1c \geqslant 1c⩾1 并且 T(m)⩽cmlg⁡mT(m) \leqslant c m \lg mT(m)⩽cmlgm 对于所有 m<nm < nm<n 都成立时，T(n)⩽cnlg⁡nT(n) \leqslant c n \lg nT(n)⩽cnlgn 也成立。只差证明一个边界情况，数学归纳法就完成了。

可惜的是，T(1)=1T(1) = 1T(1)=1，而 c1lg⁡1=0c 1 \lg 1 = 0c1lg1=0，所以 T(1)⩽c1lg⁡1T(1) \leqslant c 1 \lg 1T(1)⩽c1lg1 不成立。但是没关系，回忆一下渐近的定义，只要找到一个 n0n_0n0​ 使 n⩾n0n \geqslant n_0n⩾n0​ 时成立即可。所以让 T(2)T(2)T(2) 作为边界情况就可以了，只要选择 c⩾2c \geqslant 2c⩾2，T(2)=4⩽c2lg⁡2=2cT(2) = 4 \leqslant c 2 \lg 2 = 2 cT(2)=4⩽c2lg2=2c 就能够满足。综上，T(n)=O(nlg⁡n)T(n) = O(n \lg n)T(n)=O(nlgn) 得证。

没有轻松猜出解形式的方法，能够依靠的只有经验和运气。有时你猜对了解的形式，但是找不到合适的替换方法来使用数学归纳法。替换法很直接，但是由于这些原因不好用。

# 4.4 递归树法

在递归树中，每个节点代表了递归调用中一个子问题花费的时间。先算出每一层的总时间花费，然后算出整棵树的总时间花费。

递归树法比较简单，而且适合多种场景。有时，递归树法可以为替换法提供一些思路。在分析归并排序时使用了递归树法，这里不再多做介绍了。

# 主定理
