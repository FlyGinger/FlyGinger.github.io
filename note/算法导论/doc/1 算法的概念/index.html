# 算法导论1 算法的概念

通俗地讲，算法（algorithm）是以值或一组值作为输入（input），输出（output）值或一组值的计算步骤，也可以把算法看作解决计算问题（computational problem）的工具。举例来说，排序问题（sorting problem）的规范定义是：

输入：含有 nnn 个数字的序列 ⟨a1,a2,…,an⟩\lang a_1, a_2, \dots, a_n \rang⟨a1​,a2​,…,an​⟩。

输出：输入序列的某个满足 a1′⩽a2′⩽⋯⩽an′a'_1 \leqslant a'_2 \leqslant \dots \leqslant a'_na1′​⩽a2′​⩽⋯⩽an′​ 的排列 ⟨a1′,a2′,…,an′⟩\lang a'_1, a'_2, \dots, a'_n \rang⟨a1′​,a2′​,…,an′​⟩。

当向排序算法输入 ⟨31,41,59,26,41,58⟩\lang 31, 41, 59, 26, 41, 58 \rang⟨31,41,59,26,41,58⟩ 时，它应该输出 ⟨26,31,41,41,58,59⟩\lang26, 31, 41, 41, 58, 59 \rang⟨26,31,41,41,58,59⟩。其中，输入序列 ⟨31,41,59,26,41,58⟩\lang31, 41, 59, 26, 41, 58 \rang⟨31,41,59,26,41,58⟩ 称为排序问题的一个实例（instance）。

当对于每个输入实例，算法都能停止并返回正确结果时，算法就是正确（correct）的。正确的算法解决（solve）了给定的计算问题。不正确的算法可能在某些输入实例上不能停止，或者虽然停止但返回了错误的结果。

数据结构（data structure）能够存储和组织数据，使其便于访问和修改。没有万能的数据结构，因此了解每种数据结构的优点和缺点十分重要。

解决同一个问题的不同算法往往在效率上有巨大差异。以两个排序算法举例，插入排序（insertion sort）可以在大约 c1n2c_1 n^2c1​n2 步中排序 nnn 个元素，其中 c1c_1c1​ 是与 nnn 无关的常数。而归并排序（merge sort）可以在大约 c2nlg⁡nc_2 n \lg nc2​nlgn 步中排序 nnn 个元素，其中 lg⁡n\lg nlgn 是 log⁡2n\log_2 nlog2​n 的缩写，c2c_2c2​ 是与 nnn 无关的常数。插入排序通常有更小的常数，即 c1<c2c_1 < c_2c1​<c2​。虽然在小规模的输入实例上，插入排序更快一些，但是当输入实例的规模增长时，归并排序会反超。这是因为 n=1000n = 1000n=1000 时， lg⁡n\lg nlgn 只有大约 101010；而 n=1000000n = 1000000n=1000000 时，lg⁡n\lg nlgn 只有大约 202020。当 nnn 不断增长，最终会有 c1n2>c2nlg⁡nc_1 n^2 > c_2 n \lg nc1​n2>c2​nlgn。
