# 算法导论9 中位数和顺序统计

第 iii 位顺序统计是指 nnn 个元素的集合中第 iii 小的元素。那么元素集合的最小值（minimum）就是第一个顺序统计（i=1i = 1i=1），最大值（maximum）就是第 nnn 个顺序统计（i=ni = ni=n）。

中位数（median）是集合中位于“中间”的元素。当 nnn 为奇数时，中位数是第 i=(n+1)/2i = (n + 1) / 2i=(n+1)/2 个顺序统计；当 nnn 为偶数时，中位数是第 i=n/2i = n / 2i=n/2 和第 i=n/2+1i = n / 2 + 1i=n/2+1 两个顺序统计的平均数。通常把 i=⌊(n+1)/2⌋i = \lfloor (n + 1) / 2 \rfloori=⌊(n+1)/2⌋ 称为下中位数（lower median），把i=⌈(n+1)/2⌉i = \lceil (n + 1) / 2 \rceili=⌈(n+1)/2⌉ 称为上中位数（upper median）。为简化描述，下文中的中位数均指下中位数。

定义选择问题（selection problem）：

输入：nnn 个互不相同的数字组成的集合 AAA 和整数 iii，且有 1⩽i⩽n1 \leqslant i \leqslant n1⩽i⩽n。

输出：元素 x∈Ax \in Ax∈A，且 xxx 恰好大于 AAA 中的 i−1i - 1i−1 个元素。

# 9.1 最小值和最大值

在 nnn 个元素构成的集合中选出最小值需要至少多少次比较？容易得到比较次数的上界 n−1n - 1n−1，伪代码如下：

MINIMUM(A)
  min = A[1]
  for i = 2 to A.length
    if min > A[i]
      min = A[i]
  return min

# 同时获取最小值和最大值

可以直接把 MAXIMUM(A) 与 MINIMUM(A) 循环中的步骤组合，得到同时获取两个最值的函数。为了得到结果，一共会进行 2n−22 n - 22n−2 次比较。

实际上存在只需要 3n/23 n / 23n/2 次比较的算法。在新算法中，循环时同样需要维护最大值和最小值。每次读入两个元素，先比较这个两个元素的大小，然后将较小的与最小值比较，将较大的与最大值比较。这样一来，新算法就花费了 3 次比较处理了 2 个元素；而原先的算法需要 4 次比较来处理 2 个元素。

# 9.2 期望线性时间的选择算法

更加一般的选择问题比最值问题困难得多，但是二者的渐近运行时间都是 Θ(n)\Theta(n)Θ(n)。改造第七章中的 RANDOMIZED-QUICKSORT 可以得到 RANDOMIZED-SELECT 算法，当输入的元素各不相同时，其期望运行时间是 Θ(n)\Theta(n)Θ(n)。

RANDOMIZED-SELECT(A, p, r, i)
  if p == r
    return A[p]
  q = RANDOMIZED-PARTITION(A, p, r)
  k = q - p + 1
  if i == k
    return A[q]
  else if i < k
    return RANDOMIZED-SELECT(A, p, q - 1, i)
  else
    return RANDOMIZED-SELECT(A, q + 1, r, i - k)

# 9.3 最坏情况线性时间的选择算法

接下来介绍一种能在最坏情况运行时间 O(n)O(n)O(n) 内完成的选择算法 SELECT。与 RANDOMIZED-SELECT 不同，SELECT 是确定性算法。SELECT 算法从 nnn 个元素中选择第 iii 小的元素的基本步骤如下：

  1. 将 nnn 个元素拆分为 ⌈n/5⌉\lceil n / 5 \rceil⌈n/5⌉ 组，每组 555 个元素，最后一组可能不足 555 个。
  2. 使用插入排序将 ⌈n/5⌉\lceil n / 5 \rceil⌈n/5⌉ 组元素各自排序，然后从排序后的列表中选出中位数。
  3. 使用 SELECT 算法从 ⌈n/5⌉\lceil n / 5 \rceil⌈n/5⌉ 个中位数中选出中位数，并记为 xxx。
  4. 使用中位数的中位数 xxx 作为轴，把数组划分为两部分。假设左边的数组中有 k−1k - 1k−1 个元素，那么 xxx 就是第 kkk 小的元素。
  5. 如果 i=ki = ki=k，那么返回 xxx。否则根据 iii 的大小，递归地在左边数组或右边数组执行 SELECT。

为了确定算法的运行时间，首先需要计算比 xxx 大的元素的数量的下界。根据算法，xxx 是中位数的中位数，因此至少有一半的中位数比 xxx 大；每个中位数又代表了一个含有 555 个元素的数组，因此 ⌈n/5⌉\lceil n / 5 \rceil⌈n/5⌉ 个数组中至少有一半的数组贡献了 333 个比 xxx 大的元素。如果不算上最后一个不够 555 个元素的数组和包含 xxx 的数组，那么一共有：
