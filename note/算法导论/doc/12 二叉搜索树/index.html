# 算法导论12 二叉搜索树

# 12.1 二叉搜索树

以下是两棵二叉搜索树：

二叉搜索树（BST，binary search tree）满足以下二叉搜索树性质（binary-search-tree property）：

令 xxx 是二叉搜索树中的一个节点。如果 yyy 是 xxx 的左孩子，那么 y.key⩽x.keyy.key \leqslant x.keyy.key⩽x.key。如果 yyy 是 xxx 的右孩子，那么 y.key⩾x.keyy.key \geqslant x.keyy.key⩾x.key。

二叉搜索树的性质提供了按从小到大顺序输出树中数据的便捷方法：中序遍历（inorder tree walk）。中序遍历这个名字来自于它的具体步骤，它会先输出左子树中的数据，然后输出本节点内的数据，最后输出右子树中的数据。由于本节点的数据位于中间，所以称为中序遍历。类似地还有前序遍历（preorder tree walk）和后序遍历（postorder tree walk）。前序遍历会先输出本节点中的数据，然后是左子树和右子树中的数据。后序遍历会先输出左子树和右子树中的数据，最后输出本节点中的数据。

INORDER-TREE-WALK(x)
  if x != nil
    INORDER-TREE-WALK(x.left)
    print x.key
    INORDER-TREE-WALK(x.right)

# 12.2 二叉搜索树的查询

在二叉搜索树中，除了 SEARCH，还有 MAXIMUM、MINIMUM、SUCCESSOR、PREDECESSOR 等操作。在高度为 hhh 的二叉搜索树中，这些操作的运行时间都是 O(h)O(h)O(h)。

# 搜索

TREE-SEARCH(x, k)
  if x == nil or k == x.key
    return x
  if k < x.key
    return TREE-SEARCH(x.left, k)
  else
    return TREE-SEARCH(x.right, k)

ITERATIVE-TREE-SEARCH(x, k):
  while x != nil and k != x.key
    if k < x.key
      x = k.left
    else
      x = k.right
  return x

以上分别是递归版本和迭代版本。通常来说，迭代版本会更快一些。由于在每次迭代中都会移动到高度更低的节点，因此在高度为 hhh 的二叉搜索树中搜索的运行时间不会超过 O(h)O(h)O(h)。算法的流程十分直觉，根据二叉搜索树的性质，比 x.key 小的 k 只能在其左子树中，比 x.key 大的 k 只能在其右子树中。

# 最大值和最小值

TREE-MINIMUM(x)

    x = x.left
  return x

TREE-MAXIMUM(x)
  while x.right != nil
    x = x.right
  return x

# 先导和后继

在本章的开头提到，每个二叉搜索树都可以通过中序遍历转换成一个有序序列。节点 x 的先导（predecessor）指的是该有序序列中 x 的前一个节点，后继（successor）指的是该有序序列中 x 的后一个节点。如果二叉搜索树中保存的数据没有重复值，那么节点 x 的先导就是比它小的最大值，后继就是比它大的最小值。当然，先导和后继可能是不存在的。

TREE-SUCCESSOR(x)
  if x.right != nil
    return MINIMUM(x.right)
  y = x.parent
  while y != nil and x == y.right
    x = y
    y = y.parent
  return y

# 12.3 二叉搜索树的插入和删除

二叉搜索树中的插入比较简单，但是删除有些复杂。

# 插入

插入与搜索类似，只不过在插入算法中，只有遇到空位才会停止。

TREE-INSERT(T, z)
  // 因为以下循环停止时 x 是 nil，因此需要用 y 记录其父节点
  y = nil
  x = T.root
  while x != nil
    y = x
    if z.key < x.key
      x = x.left
    else
      x = x.right
  
  // 将 z 连接到树中
  z.p = y
  if y == nil
    // T 是空的
    T.root = z
  else if z.key < y.key
    y.left = z
  else
    y.right = z

# 删除

删除的复杂之处在于删掉节点后仍需保持二叉搜索树性质成立。假设需要删除的节点是 z，大致情况可以分为以下三种：

  1. z 没有子节点，那么可以直接删除 z。
  2. z 只有一个子节点，那么用它来替换 z 的位置。
  3. z 有两个子节点，那么用 z 的后继来替换 z 的位置。

在实际实现时会与以上情况分类有所不同，细节如下：

  1. 如果 z 没有左孩子，那么用 z 的右孩子来替换它的位置。注意右孩子可能是 nil。
  2. 如果 z 只有左孩子，那么用 z 的左孩子来替换它的位置。
  3. 如果 z 既有左孩子又有右孩子，那么找到 z 的后继 y。由于此时 z 有右孩子，所以 y 必位于 z 的右子树。注意，y 不可能有左孩子，否则 z 的后继不可能是 y，以下两种情况中都会用到这个条件。现在又可以分为两种情况：
    1. y 恰好是 z 的右孩子，那么就可以简单地令 z.key = y.key，然后 z.right = y.right 即可。
    2. y 不是 z 的右孩子，需要先令 z.key = y.key，然后再令 y.parent.left = y.right。因为 y 是 z 的后继，所以 y 一定是其父节点的左孩子。

删除本章开头的二叉搜索树中的 6 对应情况 3.1，删除 7 对应情况 3.2。

为了让删除的算法更简洁，先实现一个 TRANSPLANT 的小函数，它可以用一个子树替换某子树的位置。

TRANSPLANT(T, u, v)
  // 用 v 替换 u
  if u.parent == nil
    // u 是根节点
    T.root = v
  else if u = u.parent.left
    u.parent.left = v
  else
    u.parent.right = v
  
  if v != nil
    v.parent = u.parent

TREE-DELETE(T, z)
  if z.left == nil
    TRANSPLANT(T, z, z.right)
  else if z.right == nil
    TRANSPLANT(T, z, z.left)
  else
    y = TREE-SUCCESSOR(T, z)
    if y.parent != z
      TRANSPLANT(T, y, y.right)
      y.right = z.right
      y.right.parent = y
    TRANSPLANT(T, z, y)
    y.left = z.left
    y.left.parent = y

# 12.4 随机构造二叉搜索树

含有 nnn 个节点的二叉搜索树的高度 h⩾⌊lg⁡n⌋h \geqslant \lfloor \lg n \rfloorh⩾⌊lgn⌋。而且如果按照严格降序或者升序地插入新数据，二叉搜索树的高度会变成 h=nh = nh=n，其性能也就退化到了链表的水平了。
