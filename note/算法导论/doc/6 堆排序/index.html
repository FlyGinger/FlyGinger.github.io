# 算法导论6 堆排序

# 6.1 堆

二叉堆（binary heap）是一种可以看成完全二叉树的数组，一般简称为堆。满二叉树是满足以下两个条件的二叉树：

  • 所有叶节点均位于同一高度。
  • 所有非叶节点都有两个子节点。

把满二叉树中的节点按层序遍历的顺序编号，那么对于任意整数 k⩾0k \geqslant 0k⩾0，满二叉树的前 kkk 个节点构成一颗完全二叉树。下面展示了一个最大堆，上半部分是实际的数组结构，下半部分是堆对应的树。

二叉堆分为两种，最大堆（max-heap）和最小堆（min-heap）。两种类型中，堆的节点中的数值满足堆性质（heap property）。以最大堆为例，最大堆性质是指对于任意节点 i（除了根节点）：

A[PARENT(i)] >= A[i]
1

换句话说，节点中的值不会超过其父节点。所以最大堆中的最大元素一定位于根节点。最小堆的性质刚好相反，对于任意节点 i（除了根节点）：

A[PARENT(i)] <= A[i]
1

节点的高度（height）是从该节点出发，到达叶节点的最长简单路径中边的数量。举个例子，上文中堆的 5 号节点的高度是 111。堆的高度等于根节点的高度。由于堆符合完全二叉树相关性质，所以含有 nnn 个节点的堆的高度为 Θ(lg⁡n)\Theta(\lg n)Θ(lgn)

# 6.2 维护堆性质

可以设计一个单独的函数 MAX-HEAPIFY(A, i) 来维护堆性质。MAX-HEAPIFY 被调用时，以 LEFT(i)、RIGHT(i) 为根节点的子树必须已经是最大堆，节点 i 可能小于它的子节点。MAX-HEAPIFY 会把 i 向下移动，从而维护堆性质。

MAX-HEAPIFY(A, i)
  l = LEFT(i)
  r = RIGHT(i)

  // 如果存在左子节点，那么选出本节点与左子节点中较大的
  if l <= A.heap-size and A[l] > A[i]
    largest = l
  else
    largest = i
  
  // 如果存在右子节点，那么选出 largest 与右子节点中较大的
  if r <= A.heap-size and A[r] > A[largest]
    largest = r
  
  // 如果本节点不是最大的，那么将其和较大的子节点交换位置
  if largest != i
    交换 A[i] 和 A[largest]
    // 继续检查
    MAX-HEAPIFY(A, largest)

# 6.3 构建堆

假设堆中一共有 nnn 个节点，那么最后一个（按层序遍历）的非叶节点编号是 n/2n / 2n/2。证明如下：假设 n/2+1n / 2 + 1n/2+1 有子节点，那么它必有左子节点。那么它左子节点的编号应该是 2(n/2+1)=n+2>n2(n / 2 + 1) = n + 2 > n2(n/2+1)=n+2>n，与堆中仅有 nnn 个节点矛盾，所以 n/2+1n / 2 + 1n/2+1 没有子节点。

从节点 n/2n / 2n/2 开始，到节点 000 结束，调用 MAX-HEAPIFY，即可完成堆构建。

BUILD-MAX-HEAP(A)
  A.heap-size = A.length
  for i = A.length / 2 downto 1
    MAX-HEAPIFY(A, i)

# 6.4 堆排序

利用 MAX-HEAPIFY(i) 和 BUILD-MAX-HEAP 可以构造出堆排序算法。

HEAPSORT(A)
  BUILD-MAX-HEAP(A)
  for i = A.length downto 2
    交换 A[1] 和 A[i]
    A.heap-size = A.heap-size - 1
    MAX-HEAPIFY(A, 1)

# 6.5 优先队列

优先队列（priority queue）是一种与堆类似的数据结构，以最大优先队列为例，它提供以下操作：

  • INSERT(S, x) 向队列中插入新元素。
  • MAXIMUM(S) 返回队列中最大的元素。
  • EXTRACT-MAX(S) 返回并从队列中删除最大的元素。
  • INCREEASE-KEY(S, x, k) 把元素 x 的值增加到 k，k 必须大于等于元素 x 现在的值。

借助堆可以很方便地实现优先队列。

HEAP-MAXIMUM(A)
  // 如果堆已经为空，那么报错
  if A.heap-size < 1
    error "heap underflow"
  return A[1]
