# 算法导论7 快速排序

快速排序（quicksort）的最坏情况运行时间是 Θ(n2)\Theta(n^2)Θ(n2)。虽然如此，快速排序在实际场景中经常是表现最好的，它的期望运行时间是 Θ(nlg⁡n)\Theta(n \lg n)Θ(nlgn)，并且隐含的常数相对小。另外，快速排序还有原地排序的优点。

# 7.1 快速排序

快速排序与归并排序类似，都使用了分治策略。以下是快速排序 A[p..r] 的分治步骤：

  1. 拆分：将 A[p..r] 分成两部分 A[p..q - 1] 和 A[q + 1..r]，其中 A[p..q - 1] 的所有元素都小于等于 A[q]，A[q + 1..r] 的所有元素都大于等于 A[q]。寻找恰当的 A[q] 也是这一步的一部分。
  2. 处理：调用快速排序递归地排序 A[p..q - 1] 和 A[q + 1..r]。
  3. 合并：这一步没有工作要做。

以下是快速排序的伪代码：

QUICKSORT(A, p, r)
  if p < r
    q = PARTITION(A, p, r)
    QUICKSORT(A, p, q - 1)
    QUICKSORT(A, q + 1, r)

# 拆分数组

PARTITION 可以原地把 A[p..r] 拆分成两部分：

PARTITION(A, p, r)
  x = A[r]
  i = p - 1
  for j = p to r - 1
    if A[i] <= x
      i = i + 1
      交换 A[i] 和 A[j]
  交换 A[i + 1] 和 A[r]
  return i + 1

# 7.2 快速排序的性能

快速排序的性能取决于数组拆分是否平衡。平衡拆分使得快速排序的渐近速度和归并排序一样快，不平衡的拆分使得快速排序的渐近速度退化到与插入排序类似。

# 最坏拆分

最坏拆分发生在一个子数组有 n−1n - 1n−1 个元素，而另一个子数组没有元素时。拆分数组花费 Θ(n)\Theta(n)Θ(n)，递推式为：

T(n)=T(n−1)+T(0)+Θ(n)=T(n−1)+Θ(n)\begin{aligned} T(n) & = T(n - 1) + T(0) + \Theta(n) \\ & = T(n - 1) + \Theta(n) \\ \end{aligned} T(n)​=T(n−1)+T(0)+Θ(n)=T(n−1)+Θ(n)​

如果直接展开这个递推式：

T(n)=T(n−1)+Θ(n)=T(n−2)+Θ(n−1)+Θ(n)…=Θ(1)+Θ(2)+⋯+Θ(n)=Θ(n2)\begin{aligned} T(n) & = T(n - 1) + \Theta(n) \\ & = T(n - 2) + \Theta(n - 1) + \Theta(n) \\ & \dots \\ & = \Theta(1) + \Theta(2) + \dots + \Theta(n) \\ & = \Theta(n^2) \end{aligned} T(n)​=T(n−1)+Θ(n)=T(n−2)+Θ(n−1)+Θ(n)…=Θ(1)+Θ(2)+⋯+Θ(n)=Θ(n2)​

如果想要更严格一些，也可以使用替换法。

# 最好拆分

最好拆分发生在正好将数组平均拆分时。拆分数组花费 Θ(n)\Theta(n)Θ(n)，递推式是：

T(n)=T(n/2)+Θ(n)T(n) = T(n / 2) + \Theta(n) T(n)=T(n/2)+Θ(n)

根据主定理（情况 2），T(n)=Θ(nlg⁡n)T(n) = \Theta(n \lg n)T(n)=Θ(nlgn)。

# 平衡拆分

快速排序的平均情况运行时间更接近最好情况运行时间。假设有输入实例，使得每次拆分都十分不平衡。比如说，每次拆分都是拆成 999999 比 111 的两部分。画出递归树之后可以发现，每一层的总和都不超过 cnc ncn，树的深度是 log⁡100/99n\log_{100 / 99} nlog100/99​n。

T(n)=T(99n/100)+T(n/100)+cn=O(cnlog⁡100/99n)=O(nlg⁡n)\begin{aligned} T(n) & = T(99 n / 100) + T(n / 100) + c n \\ & = O(c n \log_{100 / 99} n) \\ & = O(n \lg n) \end{aligned} T(n)​=T(99n/100)+T(n/100)+cn=O(cnlog100/99​n)=O(nlgn)​

可以发现，对于是否平衡的判断标准还是比较“宽容”的。

# 7.3 随机快速排序

为了避免在每次拆分时都遇到最坏情况，可以把快速排序改成随机算法。

RANDOMIZED-PARTITION(A, p, r)
  i = RANDOM(p, r)
  交换 A[i] 和 A[r]
  return PARTITION(A, p, r)

# 7.4 快速排序的分析

# 最坏情况分析

在上文中证明了快速排序的最坏情况运行时间是 Θ(n2)\Theta(n^2)Θ(n2)，这里再证明一下为什么最坏情况运行时间发生在一个子数组有 n−1n - 1n−1 个元素，而另一个子数组没有元素时。

T(n)=max⁡0⩽q⩽n−1(T(q)+T(n−q−1))+Θ(n)T(n)=max⁡0⩽q⩽n−1(cq2+c(n−q−1)2)+Θ(n)\begin{aligned} T(n) & = \max_{0 \leqslant q \leqslant n - 1} (T(q) + T(n - q - 1)) + \Theta(n) \\ T(n) & = \max_{0 \leqslant q \leqslant n - 1} (c q^2 + c(n - q - 1)^2) + \Theta(n) \end{aligned} T(n)T(n)​=0⩽q⩽n−1max​(T(q)+T(n−q−1))+Θ(n)=0⩽q⩽n−1max​(cq2+c(n−q−1)2)+Θ(n)​

猜测 T(n)=Θ(n2)T(n) = \Theta(n^2)T(n)=Θ(n2) 然后代入，得到了上面的第二个式子。容易知道，在 0⩽q⩽n−10 \leqslant q \leqslant n - 10⩽q⩽n−1 时，cq2+c(n−q−1)2c q^2 + c(n - q - 1)^2cq2+c(n−q−1)2 的最大值位于 q=0q = 0q=0 和 q=n−1q = n - 1q=n−1 处。

# 平均情况分析

随机快速排序的期望运行时间应该和快速排序的平均情况运行时间相等，所以这里仅分析一下随机快速排序的期望运行时间。由于随机快速排序与快速排序仅有拆分时选择轴的部分有所不同，其余部分全部相同，所以主要讨论拆分函数。

快速排序的运行时间主要取决于拆分函数。在拆分函数中，算法选择的轴不会出现在后续的递归调用中，因此拆分函数最多被调用 nnn 次。
